{\footnotesize\ttfamily void Ink.\+Runtime.\+Story.\+Choose\+Path\+String (\begin{DoxyParamCaption}\item[{string}]{path,  }\item[{bool}]{reset\+Callstack = {\ttfamily true},  }\item[{params object\mbox{[}$\,$\mbox{]}}]{arguments }\end{DoxyParamCaption})\hspace{0.3cm}{\ttfamily [inline]}}



Change the current position of the story to the given path. From here you can call \mbox{\hyperlink{class_ink_1_1_runtime_1_1_story_a92d12892609d28c1b21f6cd809a4c643}{Continue()}} to evaluate the next line. 

The path string is a dot-\/separated path as used internally by the engine. These examples should work\+:

my\+Knot my\+Knot.\+my\+Stitch

Note however that this won\textquotesingle{}t necessarily work\+:

my\+Knot.\+my\+Stitch.\+my\+Labelled\+Choice

...because of the way that content is nested within a weave structure.

By default this will reset the callstack beforehand, which means that any tunnels, threads or functions you were in at the time of calling will be discarded. This is different from the behaviour of Choose\+Choice\+Index, which will always keep the callstack, since the choices are known to come from the correct state, and known their source thread.

You have the option of passing false to the reset\+Callstack parameter if you don\textquotesingle{}t want this behaviour, and will leave any active threads, tunnels or function calls in-\/tact.

This is potentially dangerous! If you\textquotesingle{}re in the middle of a tunnel, it\textquotesingle{}ll redirect only the inner-\/most tunnel, meaning that when you tunnel-\/return using \textquotesingle{}-\/\texorpdfstring{$>$}{>}-\/\texorpdfstring{$>$}{>}\textquotesingle{}, it\textquotesingle{}ll return to where you were before. This may be what you want though. However, if you\textquotesingle{}re in the middle of a function, Choose\+Path\+String will throw an exception.
